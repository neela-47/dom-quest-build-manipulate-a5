Important FAQ

What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a webpage as a tree of objects, where each object corresponds to a part of the document (such as elements, attributes, and text). The DOM allows scripts (like JavaScript) to interact with, modify, and update the content, structure, and style of a webpage dynamically, without needing to reload the page.

How do you select an element from the DOM?

To select an element from the DOM, JavaScript provides several methods. The most common ways include: 1. getElementById()
2. getElementsByClassName()
3. getElementsByTagName()
4. querySelector()
5. querySelectorAll()

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM that allows you to handle events more efficiently by attaching a single event listener to a parent element rather than multiple listeners to individual child elements. The event listener on the parent element can respond to events triggered by its child elements through a mechanism called event bubbling.

How do you manipulate an element's attributes and styles using the DOM?

To manipulate an element's attributes and styles using the DOM, you can use methods like setAttribute(), getAttribute(), or directly access properties like id, className, or src for attributes. For styles, you can modify the style property of an element to change its CSS inline. For example, to change an image's src and set its width via CSS, you could do: image.setAttribute('src', 'newImage.jpg'); and image.style.width = '200px';. You can also toggle classes using classList.add(), remove(), or toggle() for more advanced styling control.
